REMEMBER THIS
Follow the destination.

When picking a log streaming method, don't start by asking "what am I streaming?" — start by asking "where does the SIEM live?" The destination dictates the method.

IF ON-PREM
Bridge into the customer network
ZIA LOGS
NSS
Nanolog Streaming Service — customer-deployed VM makes outbound TLS to the Nanolog cluster, forwards to the local SIEM.
ZPA LOGS
LSS
Log Streaming Service — piggybacks on the existing App Connector infrastructure to reach the on-prem SIEM.
IF CLOUD
Wire the two clouds together
ZIA OR ZPA
Cloud-to-Cloud Log Streaming
Direct HTTPS from the Zscaler cloud to the SIEM cloud — no customer VM, no App Connector. Works with Splunk Cloud, Sumo Logic, Devo, and any SIEM that accepts a customisable HTTPS outbound connector.

WORTH REMEMBERING
★
MENTAL NOTE
NSS is customer-managed. Zscaler provides the VM image, but the customer owns patching, monitoring, and lifecycle. It's a normal Linux VM with a Zscaler agent — not a fully managed appliance like a PSE.
★
MENTAL NOTE
Both NSS and LSS use outbound-only connections. No inbound firewall rules needed on the customer side — same design principle as App Connectors. This is Zscaler's signature architectural pattern.
→
EXAM TIP
Watch for distractors that swap NSS ↔ LSS. The exam frequently tests "which service streams ZPA logs to the customer's on-prem Splunk?" — the answer is LSS, not NSS.
→
EXAM TIP
Cloud-to-Cloud is not limited to Splunk Cloud, Sumo, and Devo — it supports any SIEM reachable via a customisable HTTPS outbound connector. Don't fall for the "only three vendors" distractor.
★
MENTAL NOTE
NSS can buffer up to 1 hour of logs during connectivity issues with the Nanolog cluster — useful for scenario questions about temporary network outages.
